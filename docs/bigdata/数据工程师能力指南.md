# 大数据工程师能力指南
## 一、知识点一
1、分布式系统的概率和理论，比如CAP、BASE、FLP、sharding和replicate机制
Base 理论是对CAP中一致性和可用性的权衡结果, 最终思想是无法做到强一制性，但每一个应用都可以根据自身业务特点，采取适当的方式来到达最终一致性。
FLP 不可能原理：它指出在最小化异步网络通信场景下，即使只有一个节点失败，也没有一种确定性的共识算法能够使得在其他节点之间保持数据一致。
